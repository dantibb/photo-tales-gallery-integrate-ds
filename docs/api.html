GET /api/media/{id}/preview

Get image preview (serves the actual image file)

Parameters

  • id (string, required) - Media item document ID

Response

Binary image file with appropriate Content-Type header

PUT /api/media/{id}/description

Update the description of a media item

Parameters

  • id (string, required) - Media item document ID

Request Body

{
  "description": "Updated description of the photo"
}

Response

{
  "success": true,
  "message": "Description updated successfully"
}

AI Features

POST /api/media/{id}/generate-summary

Generate AI summary for a media item

Parameters

  • id (string, required) - Media item document ID

Request Body

{}

Response

{
  "summary": "A beautiful sunset over the ocean",
  "title": "Ocean Sunset",
  "summary_text": "This photo captures a stunning sunset over the ocean with vibrant orange and pink hues reflecting off the water."
}
POST /api/media/{id}/generate-tags

Generate AI tags for a media item

Parameters

  • id (string, required) - Media item document ID

Response

{
  "tags": ["sunset", "ocean", "nature", "landscape", "golden hour"],
  "message": "Tags generated successfully"
}
POST /api/ai-photographer/summary

Generate photographer summary based on tag

Request Body

{
  "tag": "vacation",
  "model": "gpt-4o-mini"
}

Response

{
  "summary": "I love taking photos during vacations, especially of landscapes and sunsets.",
  "photo_count": 15,
  "tag": "vacation"
}
POST /api/generate-photographer-summary

Generate conversational AI response for photographer

Request Body

{
  "tag": "vacation",
  "userMessage": "Tell me about this photo",
  "conversationHistory": [
    {"role": "user", "content": "Hello"},
    {"role": "assistant", "content": "Hi! I'm your AI assistant."}
  ],
  "model": "gpt-4o-mini"
}

Response

{
  "summary": "Oh yeah, that was a great day - remember how excited we were?",
  "tag": "vacation",
  "photo_count": 15
}

Interview System

POST /api/media/{id}/interview/start

Start an AI interview for a media item

Parameters

  • id (string, required) - Media item document ID

Response

{
  "media_id": "doc123",
  "ai_question": "Where was this photo taken?",
  "messages": [
    {
      "role": "system",
      "content": "You are Memory Gatherer..."
    },
    {
      "role": "assistant",
      "content": "Where was this photo taken?"
    }
  ]
}
POST /api/media/{id}/interview/chat

Continue an AI interview conversation

Parameters

  • id (string, required) - Media item document ID

Request Body

{
  "user_text": "It was taken in Hawaii during our vacation",
  "messages": [
    {
      "role": "system",
      "content": "You are Memory Gatherer..."
    },
    {
      "role": "assistant",
      "content": "Where was this photo taken?"
    },
    {
      "role": "user",
      "content": "It was taken in Hawaii during our vacation"
    }
  ]
}

Response

{
  "media_id": "doc123",
  "ai_question": "That sounds wonderful! What was the best part of your Hawaii vacation?",
  "messages": [
    // ... previous messages plus new assistant message
  ]
}
POST /api/media/{id}/interview/save

Save interview conversation as context

Parameters

  • id (string, required) - Media item document ID

Request Body

{
  "messages": [
    {
      "role": "assistant",
      "content": "Where was this photo taken?"
    },
    {
      "role": "user",
      "content": "It was taken in Hawaii during our vacation"
    }
  ]
}

Response

{
  "context_id": "ctx123",
  "message": "Interview saved successfully"
}

Error Responses

Common Error Codes

400 Bad Request

Invalid request parameters or missing required fields

{
  "error": "Missing required field: description"
}

404 Not Found

Resource not found

{
  "error": "Media item not found"
}

500 Internal Server Error

Server error or external service failure

{
  "error": "Failed to generate summary: OpenAI API error"
}

Rate Limiting

Rate Limits: AI endpoints are subject to OpenAI API rate limits. The system implements basic rate limiting to prevent abuse and manage costs.

Frontend Integration

TypeScript Interfaces

The frontend uses these TypeScript interfaces for type safety:

export interface MediaItem {
  id: string;
  file_path: string;
  filename: string;
  title?: string;
  summary?: string;
  tags?: string[];
  file_size?: number;
  file_type?: string;
  created_at: string;
  updated_at: string;
  contexts?: Context[];
  image_metadata?: any;
}

export interface Context {
  id: string;
  text: string;
  context_type?: string;
  created_at: string;
}

export interface InterviewMessage {
  role: 'system' | 'user' | 'assistant';
  content?: string;
  type?: 'text' | 'image_url';
  data?: string;
}

API Benefits: The RESTful API design provides a clean, consistent interface for all operations. All endpoints return JSON responses and follow standard HTTP status codes.
