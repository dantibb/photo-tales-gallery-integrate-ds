Installation Guide

This guide will walk you through setting up Photo Tales Gallery on your local machine for development.

Prerequisites

System Requirements

  • • Node.js: Version 18 or higher
  • • Python: Version 3.8 or higher
  • • Git: For cloning the repository
  • • Modern Browser: Chrome, Edge, or Safari

External Services

  • • OpenAI API Key: For AI features
  • • Google Cloud Account: For storage (optional)
  • • Microphone Access: For voice features

Quick Installation

One-Command Setup 
# Clone and setup everything
git clone https://github.com/your-username/photo-tales-gallery.git
cd photo-tales-gallery
./start_all.sh

The start_all.sh Script

What start_all.sh Does

Setup & Installation

  • • Installs frontend dependencies (npm install)
  • • Installs backend dependencies (pip install)
  • • Creates config.env from example if missing
  • • Validates configuration and warns about missing API keys

Server Management

  • • Starts backend server in background
  • • Starts frontend server in background
  • • Provides process IDs for easy shutdown
  • • Shows access URLs and status

Configuration Warnings

The script checks for these required environment variables and warns if they're missing:

  • • OPENAI_API_KEY - Required for AI features
  • • GCS_BUCKET_NAME - Required for cloud storage
  • • GOOGLE_APPLICATION_CREDENTIALS - Required for Google Cloud
  • • FLASK_SECRET_KEY - Required for session security

Detailed Installation Steps

1

Clone the Repository

git clone https://github.com/your-username/photo-tales-gallery.git
cd photo-tales-gallery
2

Install Frontend Dependencies

npm install

This installs React, TypeScript, Tailwind CSS, and all other frontend dependencies.

3

Install Backend Dependencies

cd backend
pip install -r requirements.txt

This installs Flask, OpenAI, and other Python dependencies.

4

Configure Environment Variables

cd backend
cp config.env.example config.env

Edit config.env with your API keys and configuration.

5

Start the Application

./start_all.sh

This comprehensive script installs dependencies, sets up configuration, and starts both servers with proper process management.

Environment Configuration

Required Environment Variables

OpenAI Configuration

# Your OpenAI API key (required for AI features)
OPENAI_API_KEY=sk-your-openai-api-key-here

Get your API key from OpenAI Platform

Google Cloud Configuration (Optional)

# Path to your Google Cloud service account JSON key file
GOOGLE_APPLICATION_CREDENTIALS=/path/to/your/service-account-key.json

# Your Google Cloud Storage bucket name
GCS_BUCKET_NAME=your-gcs-bucket-name

# Your Firestore database name (usually '(default)')
FIRESTORE_DATABASE_NAME=(default)

Only required if using Google Cloud Storage and Firestore

Flask Configuration

# Secret key for Flask sessions (generate a random string)
FLASK_SECRET_KEY=your-secret-key-here

Generate a random string for session security

Database Setup

Local Development Database

For local development, the application uses TinyDB (JSON-based) by default. No additional setup required.

Google Cloud Firestore (Production)

1. Create Google Cloud Project

  1. Go to Google Cloud Console
  2. Create a new project or select existing
  3. Enable Firestore and Cloud Storage APIs

2. Create Service Account

  1. Go to IAM & Admin > Service Accounts
  2. Create a new service account
  3. Download the JSON key file
  4. Set the path in GOOGLE_APPLICATION_CREDENTIALS

3. Create Storage Bucket

  1. Go to Cloud Storage > Buckets
  2. Create a new bucket
  3. Set the bucket name in GCS_BUCKET_NAME

Troubleshooting

Port Already in Use

If port 5001 is already in use, change the port in backend/local_api.py or kill the existing process.

OpenAI API Key Issues

Ensure your OpenAI API key is valid and has sufficient credits. Check the API key format and permissions.

Python Dependencies

If you encounter Python dependency issues, try using a virtual environment: python -m venv venv && source venv/bin/activate

Node.js Version

Ensure you're using Node.js 18 or higher. Check with node --version and update if necessary.

Verification

Check Installation

1. Frontend (http://localhost:5173)

You should see the Photo Tales Gallery interface with a photo grid.

2. Backend API (http://localhost:5001/api/health)

curl http://localhost:5001/api/health
# Should return: {"status": "healthy"}

3. AI Features

Try generating tags or starting an interview on a photo to test OpenAI integration.

4. Voice Features

Grant microphone permissions and test voice input in the chat interface.
